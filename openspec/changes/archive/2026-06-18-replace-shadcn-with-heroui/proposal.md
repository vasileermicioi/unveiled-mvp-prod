## Why

The shadcn-based primitives currently shipped in `src/components/ui/` are a transitional scaffold; the approved HeroUI Ladle design system (from iteration 11 change 1) is the new visual contract for the app. To stop shipping two parallel UI systems, the production code base must drop its shadcn dependencies and rebuild every primitive (`Button`, `Panel`, `Card`, `Badge`, `StatPanel`, `Field`, `TextInput`, `SelectInput`, `TextArea`, `Divider`, `StatePanel`, `TableShell`, `TableRow`) on top of HeroUI while keeping the public prop signatures and selectors stable so gherkin parity tests do not break.

## What Changes

- **Promote HeroUI to a production dependency.** Move `@nextui-org/react` (and any required peer packages) from `devDependencies` to `dependencies` in `package.json` and re-run `bun install` to update `bun.lock`.
- **Promote the HeroUI theme to production.** Move `src/components/ui/heroui-replica/theme.ts` to `src/lib/heroui-theme.ts` (or `src/styles/heroui-theme.ts`); update `src/components/ui/heroui-replica/provider.tsx` (and any other importer) to consume the production module. The `heroui-replica/` directory remains Ladle-only and continues to be gated by `bun run heroui-design-system-replica:check`.
- **Rebuild production primitives on HeroUI.** Replace `src/components/ui/button.tsx` and `src/components/ui/unveiled-primitives.tsx` with HeroUI-backed wrappers that preserve the existing variant/size matrix (`default`, `primary`, `secondary`, `yellow`, `active`, `copied`, `destructive`, `ghost`, `outline`, `muted`, `link` × `default`, `sm`, `lg`, `icon`, `icon-sm`) and the `loading` / `asChild` props. Add new production primitives for surfaces previously backed by Mantine shims: `Modal`, `Drawer`, `Tabs`, `Menu`, `Toast`/`Notification`.
- **Wire `HeroUIProvider` into the app shell.** Add the provider to the root client provider stack (new `src/components/providers/heroui-provider.tsx` or folded into `src/components/providers/query-provider.tsx`) and ensure SSR/Cloudflare compatibility by gating client-only APIs behind `client:only="react"` or dynamic imports.
- **Update every consumer.** Walk `src/components/unveiled/`, `src/components/payments/`, `src/components/providers/`, `src/pages/`, and `src/layouts/` to use the new HeroUI-backed primitives. Replace any remaining imports from `src/components/ui/mantine-replica/` (should be none after change 1) and any shadcn-specific patterns. Port prop-level concerns (`tone`, `shadow`, `interactive`, `state`) into HeroUI style props.
- **Clean up shadcn remnants.** Update or remove `components.json` if shadcn is no longer the component source, and drop shadcn-only utilities (`class-variance-authority` if no longer used, `tailwind-merge` if HeroUI handles variant composition, etc.) after a full consumer audit.

## Capabilities

### New Capabilities

_None._ This change replaces the implementation of existing UI surfaces; it does not add new user-visible capabilities.

### Modified Capabilities

- `app-shell`: the client root provider stack now mounts `HeroUIProvider`, and the global theme configuration is sourced from the new production theme module instead of the Ladle-only replica.
- `ui-system`: every primitive exported from `src/components/ui/` is now backed by HeroUI while preserving the Unveiled look-and-feel, the variant/size matrix, the `loading` / `asChild` props, the public prop signatures, and the selector discipline required by the gherkin parity suite.

## Implementation Outcome

The umbrella was implemented as 9 sub-changes (`replace-shadcn-with-heroui-2a` through `2i`). Notes on what actually shipped, recorded here for reviewers:

- **`Button` and `unveiled-primitives`**: the wrappers preserve the public prop surface (`variant`/`size`/`loading`/`asChild` on `Button`; `tone`/`shadow`/`interactive`/`state` on the unveiled primitives). `bun run check` is non-breaking (same 6 pre-existing `astro check` errors as the clean baseline; no new errors). 34 consumer call sites compile unchanged.
- **`SelectInput`**: the wrapper accepts the existing `<option>` children API and translates to HeroUI's `<SelectItem>` internally, so the 9 existing call sites needed no edits. HeroUI's `selectedKeys` / `onSelectionChange` are also exposed via the same wrapper for new code that prefers the native HeroUI API.
- **New primitives (`Modal` / `Drawer` / `Tabs` / `Menu` / `Toast`)**: added as purely additive primitives at `src/components/ui/modal.tsx`, `drawer.tsx`, `tabs.tsx`, `menu.tsx`, `toast.tsx`. No production call site was migrated (the production app has no `Modal`/`Drawer`/`Tabs`/`Menu`/`Toast` consumer today). The existing legacy `ModalShell` in `src/components/unveiled/app-shell.tsx:730` (a full-screen brand-yellow overlay used by `BookingModal` and the admin "Bestätigen" dialog) was intentionally **not** migrated — it is a different visual treatment (full-screen takeover, not a centered dialog). A future change can choose to migrate `ModalShell` to use the new `Modal` if the centered-dialog treatment is preferred.
- **`Menu`**: implemented on HeroUI's `Dropdown` family (the production-grade menu primitive with full keyboard navigation), not the replica's `Popover` choice. `Dropdown` is HeroUI's `Menu`.
- **`Toast`**: HeroUI does not ship a toast component. The new `toast.tsx` ships a `ToastProvider` + `useToast()` hook + `Toast` component built on HeroUI's `Alert` (matching the replica's reference design).
- **`components.json`**: removed (no shadcn scaffolding remains).
- **Package removals**: `@radix-ui/react-slot`, `class-variance-authority`, `clsx`, `tailwind-merge` were audited but **kept** in `package.json`. They are used by the new HeroUI-backed `Button` (`Slot` for `asChild`, `cva` for the 11-variant × 5-size matrix) and by `src/lib/utils.ts` (`cn()` used by every new HeroUI primitive for className overlays). The umbrella's spec intent ("shadcn packages removed") is partially met; a future refactor that drops `cva` and `cn()` from the new HeroUI primitives would unlock the package removals.
- **Test gates**: `bun run test:e2e` and `bun run test:ladle` are pre-existing-infra-broken in the baseline (every e2e scenario times out at 30s; the `ladle` Playwright project is not configured). The runnable gates — `bun run check` (6 pre-existing errors, unchanged), `bun run check:heroui-replica` (the outer replica + ladle checks pass; the inner `check` fails on the same 6 pre-existing errors), `bun run build` (green; +3s vs baseline, well within the performance budget) — are all green. The umbrella's stop-and-report policy on test failure is respected; the situation is recorded in slice `replace-shadcn-with-heroui-2i/tasks.md` and here.
