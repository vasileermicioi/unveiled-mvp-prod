# i18n dictionary and coverage gate

The application's user-facing copy lives in a single DE/EN dictionary
at `packages/api/src/i18n.ts`. Every consumer reads copy through
`copyFor(language).<dot.path>`. The dictionary and the consumers are
kept in lock-step by a CI gate (`bun run check:i18n-coverage`).

## Dictionary shape

The dictionary is exported as `appCopy = { DE: {...}, EN: {...} } as const`.
Both `DE` and `EN` MUST have an identical shape; every leaf string
MUST be present in both languages. Branches and leaves are enforced
at compile time for the typed bundles (`shell.*`, `auth.forms.*`,
`auth.errors.*`) and at runtime for everything else.

Add a new translatable string by adding the same key to both branches
of `appCopy`:

```ts
export const appCopy = {
  DE: {
    partner: {
      alreadyUsed: "Bereits genutzt",
    },
  },
  EN: {
    partner: {
      alreadyUsed: "Already used",
    },
  },
} as const;
```

## Consumer contract

Components and pages read copy through `copyFor(language)`:

- React islands under `packages/app/src/**` use the
  `useCopy()` hook from
  `packages/app/src/components/unveiled/context-primitives.tsx`.
- Astro pages call `copyFor(lang).<path>` directly from
  `packages/app/src/lib/i18n.ts`.
- The `app` package re-exports the dictionary via
  `packages/app/src/lib/i18n.ts` (a one-line re-export of
  `@unveiled/api/i18n`); consumers SHOULD import from the app
  re-export so future swaps of the dictionary module remain a
  one-file change.

**Do not** inline `selectedLanguage === "DE" ? "..." : "..."`
ternaries. They bypass the dictionary and the gate will not catch
the missing key. Use `copyFor(language).<path>` lookups instead.

## Coverage gate

`bun run check:i18n-coverage` runs
`scripts/i18n-coverage.ts`, which:

1. Walks every `.ts`, `.tsx`, and `.astro` file under
   `packages/app/src/**` and `packages/landing/src/**`.
2. Greps each file for `copyFor(<expr>).<dot.path>` access patterns
   and collects the unique paths.
3. Imports `appCopy` from `@unveiled/api/i18n` and walks the `DE`
   and `EN` branches recursively, collecting the flat leaf-path
   sets and the intermediate branch-path sets.
4. For every referenced path that is neither a leaf nor a branch in
   a given language, prints
   `i18n.coverage: missing key '<path>' in <LANG> (referenced by <file>:<line>)`
   and exits `1`.
5. When every referenced path is known, prints
   `i18n.coverage: ok — N leaf paths present in both DE and EN.`
   and exits `0`.

The gate is wired into `bun run check` (right at the start of the
umbrella command), so a missing key blocks the umbrella.

## Adding a new key

1. Add the same key to both `DE` and `EN` in
   `packages/api/src/i18n.ts`.
2. Add `copyFor(language).<your.new.path>` lookups in the
   consumers.
3. Run `bun run check:i18n-coverage` to confirm the gate stays green.
4. Run `bun run test:unit` to confirm the unit suite stays green.

## Adding a new typed bundle

The typed-parity rules for `shell.*`, `auth.forms.*`, and
`auth.errors.*` are exported as `ShellCopy`, `AuthFormCopy`, and
`AuthErrorCopy` from `packages/api/src/i18n.ts`. Adding a typed
bundle (e.g. `PartnerCopy`) is a future iteration: this change ships
a runtime gate, not a static-type upgrade for those branches.
